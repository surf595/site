# Stage 1 — mobile-first UX-каркас (выполнение цели этапа)

Документ реализует **первую цель этапа**: собрать практический mobile-first каркас сайта, который одновременно даёт быстрый путь к записи и путь осторожной проверки доверия/условий.

## 1. UX-скелет сайта (P0)

## 1.1 Карта экранов и роль каждого

| Экран | URL | Что объясняет пользователю | Основной CTA |
|---|---|---|---|
| Главная | `/` | Кто специалист, с чем работает, как начать | Записаться |
| Услуги | `/services` | Какие форматы помощи доступны | Выбрать формат |
| Индивидуальная работа | `/services/individual` | Как проходит работа 1:1, длительность, стоимость | Выбрать время |
| О специалисте | `/about` | Квалификация, подход, профессиональная рамка | Записаться |
| Границы и безопасность | `/about/boundaries` | Конфиденциальность, исключения, границы контакта | Выбрать время |
| Запись | `/booking` | Простая запись с минимальными полями и согласием | Подтвердить запись |
| Подтверждение | `/booking/confirmed` | Что делать после записи, как подготовиться | Добавить в календарь |
| Privacy | `/privacy` | Как и зачем обрабатываются данные | Вернуться к записи |
| Consent | `/consent` | Явное согласие на обработку данных | Вернуться к записи |
| Terms | `/terms` | Оплата, отмена, перенос, дедлайны | Вернуться к записи |

---

## 2. Mobile wireframe-структура P0 экранов

Ниже — структура блоков сверху вниз для мобильного первого экрана.

### 2.1 Главная (`/`)
1. Hero: спокойное позиционирование + CTA `Записаться`.
2. Trust bar: опыт/квалификация/формат (онлайн, город).
3. «С чем я работаю» (6–8 карточек запросов).
4. «Как проходит работа» (3 шага: знакомство → сессии → оценка динамики).
5. «Формат и условия» (длительность, стоимость, перенос/отмена коротко).
6. Короткий блок «О специалисте» + ссылка `Подробнее`.
7. Блок «Границы и безопасность» + ссылка `Читать рамки`.
8. Мини-FAQ (3–5 вопросов).
9. Финальный CTA `Выбрать время`.
10. Footer legal: privacy / consent / terms / contacts.

### 2.2 Услуги (`/services`)
1. Заголовок + краткое пояснение выбора.
2. Карточка `Индивидуальная работа` (P0, активная).
3. Карточки `Пары` / `Подростки` с бейджем «скоро» (P2).
4. Подсказка по выбору + CTA `Перейти к индивидуальной`.

### 2.3 Индивидуальная работа (`/services/individual`)
1. Hero услуги + CTA `Выбрать время`.
2. Для кого подходит / не подходит.
3. Как проходит первая встреча.
4. Частота/длительность/стоимость.
5. Отмена и перенос (кратко + линк на terms).
6. FAQ услуги.
7. CTA `Записаться`.

### 2.4 О специалисте (`/about`)
1. Фото/представление (не рекламный tone).
2. Образование и квалификация.
3. Подход и ценности работы.
4. С чем работаю / с чем не работаю.
5. Ссылка на `Границы и безопасность`.
6. CTA `Записаться`.

### 2.5 Границы и безопасность (`/about/boundaries`)
1. Принципы конфиденциальности.
2. Исключения (угроза жизни/закону и т.п.).
3. Границы контакта вне сессий.
4. Правила переноса/отмены (коротко).
5. Ссылки на Privacy / Consent / Terms.
6. CTA `Выбрать время`.

### 2.6 Запись (`/booking`)
1. Шапка «Запись на первую встречу».
2. Выбор формата (онлайн/очно если доступно).
3. Выбор времени (слоты/календарь).
4. Короткая форма:
   - имя или псевдоним;
   - email и/или телефон/мессенджер;
   - комментарий (опционально).
5. Чекбокс согласия + ссылки privacy/consent/terms.
6. Короткая подсказка по отмене/переносу.
7. CTA `Подтвердить запись`.
8. Состояния ошибок (валидация под полем, без сброса данных).

### 2.7 Подтверждение (`/booking/confirmed`)
1. Сообщение об успешной записи.
2. Дата/время/формат.
3. Что подготовить к первой встрече.
4. `Добавить в календарь` + `Связаться`.
5. Ссылка на перенос/отмену (terms/contact).

### 2.8 Legal-экраны (`/privacy`, `/consent`, `/terms`)
1. Короткое вступление простым языком.
2. Основные пункты (структурно, списками).
3. Фиксированная кнопка `Вернуться к записи`.

---

## 3. Ключевые пользовательские маршруты (кликабельная логика)

### Flow A — быстрая запись
`/` → `/booking` → `/booking/confirmed`

### Flow B — проверка специалиста
`/` → `/about` → `/about/boundaries` → `/booking`

### Flow C — вход через услугу
`/services` → `/services/individual` → `/booking`

### Flow D — юридическая проверка без потери записи
`/booking` → `/privacy` (или `/consent` / `/terms`) → `/booking`

**Обязательное правило:** из всех P0 экранов пользователь должен попасть к записи не более чем за 2 шага.

## 4. Навигация и сквозные элементы

### Header (mobile)
- Лого
- Бургер-меню: О специалисте, Услуги, FAQ, Контакты
- Кнопка `Запись`

### Sticky CTA (mobile)
- Фиксированная кнопка `Записаться` на P0 контентных страницах.
- На `/booking` кнопка меняется на `Подтвердить запись`.

### Footer
- Контакты
- Privacy
- Consent
- Terms

---

## 5. UX-правила снижения тревоги (обязательные)

1. Нейтральные формулировки, без давления и манипуляций.
2. Никаких «агрессивных» CTA и таймеров.
3. Минимум обязательных полей в записи.
4. Прозрачность правил до отправки формы.
5. Видимый маршрут назад из legal-страниц в запись.
6. Ошибки формы объясняются человеческим языком.

---

## 6. Definition of Done по первой цели этапа

Цель этапа считается выполненной, если:
1. Утверждены все P0 экраны из раздела 1.
2. Для каждого P0 экрана есть mobile wireframe-структура из раздела 2.
3. Реализованы/прототипированы Flows A–D.
4. На всех P0 соблюдается правило «до записи ≤ 2 шага».
5. Legal-точки встроены в маршрут записи и имеют возврат в `/booking`.

## 7. Что передаётся следующей команде (handoff минимум)

5. Список юридических ссылок и мест их показа.
